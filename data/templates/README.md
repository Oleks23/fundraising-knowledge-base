# FCC CSV Templates

These templates provide header-only CSV files for the Microsoft 365 MVP implementation of the Fundraising Command Centre.

The templates align with `docs/SHAREPOINT_LISTS_DESIGN.md` and are intended to support SharePoint List setup, CSV-based ingestion, and synthetic data generation.

No activities template is included. Activities remain in source systems such as RE NXT Actions, Salesforce Tasks, Planner, Asana, Smartsheet, and Microsoft Project. FCC consumes activity data only to generate commitments, risks, readiness indicators, compliance metrics, and metric snapshots.

## Templates

| Template | Purpose |
| --- | --- |
| `programs_template.csv` | High-level fundraising areas such as Annual Giving, Major Gifts, Events, Stewardship, and Corporate Partnerships. |
| `initiatives_template.csv` | Operational work portfolios linked to Programs. Includes source reference fields for mapped CRM, planning, or management-originated records. |
| `commitments_template.csv` | Obligations requiring follow-up or completion. Includes source reference fields where commitments originate from CRM, planning, or other source records. |
| `dependencies_template.csv` | Blockers, prerequisites, approvals, and cross-functional dependencies linked to Initiatives. |
| `risks_template.csv` | Operational risks, whether manually entered or generated by rules. Includes source reference fields for traceability. |
| `knowledge_template.csv` | Metadata for operational knowledge assets stored in the SharePoint Knowledge Library. |
| `metric_snapshots_template.csv` | Historical calculated metrics used by Power BI and AI summaries. |
| `configuration_template.csv` | Client-specific thresholds, scoring weights, escalation rules, SLA definitions, and other configurable settings. |

## Usage Notes

- Headers only are provided.
- Column names use CSV-friendly snake_case.
- Lookup fields such as `program` and `initiative` should reference the corresponding FCC record identifiers or names according to the implementation mapping.
- `source_system` and `source_record_id` are included only where source traceability is part of the SharePoint design.
- Do not use these templates to recreate source-system activity/task records inside FCC.
